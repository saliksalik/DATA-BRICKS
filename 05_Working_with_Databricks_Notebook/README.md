# Lab 5: Working with Databricks Notebook

This lab introduces Databricks Notebooks, covering creation, configuration, code execution, and key features.

## Step-by-Step Guide

### Step 1: Access Databricks Workspace Home Page
- Navigate to your Azure Databricks workspace home page.

![Databricks Workspace Home Page](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Databricks%20Workspace%20Home%20Page.png?raw=true)

### Step 2: Navigate to Workspace Menu and Create Folder
- Go to the Workspace menu, expand the folder.
- Right-click workspace directory, create a new folder, and name it.

![Workspace Menu - Creating New Folder](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Workspace%20Menu%20-%20Creating%20New%20Folder.png?raw=true)

### Step 3: Create and Rename Notebook
- Right-click the folder, create a new notebook.
- Rename it to a meaningful name.

![Creating New Notebook](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Creating%20New%20Notebook.png?raw=true)

### Step 4: View Notebook Default Settings
- See the default name, language (Python), and options to change language.

![Notebook Default View (Name and Language)](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Notebook%20Default%20View%20(Name%20and%20Language).png?raw=true)

### Step 5: Attach Notebook to Cluster
- Click the Connect button; if no cluster, create new.

![Attaching Notebook to Cluster](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Attaching%20Notebook%20to%20Cluster.png?raw=true)

### Step 6: Create New Cluster
- Choose policy, name cluster, select VM/runtime, review cost, create.

![Creating New Cluster from Notebook](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Creating%20New%20Cluster%20from%20Notebook.png?raw=true)

### Step 7: Configure Cluster Options
- Set VM size, runtime version, and check summary.

![Cluster Configuration Options](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Cluster%20Configuration%20Options.png?raw=true)

### Step 8: Write and Run Code in Cell
- Enter code in the first cell, run it (Ctrl+Enter).

![Writing and Running Code in Notebook Cell](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Writing%20and%20Running%20Code%20in%20Notebook%20Cell.png?raw=true)

### Step 9: Add Cell and Run with Output
- Add another cell, write print code, run to see output and state.

![Notebook Output and State](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Notebook%20Output%20and%20State.png?raw=true)

### Step 10: Explore Notebook Menus
- Use File (export), Edit (format), View (modes), Run (options).

![Notebook Menus and Options (File, Edit, View, Run)](https://github.com/saliksalik/DATA-BRICKS/blob/main/05_Working_with_Databricks_Notebook/outputs/Notebook%20Menus%20and%20Options%20(File%2C%20Edit%2C%20View%2C%20Run).png?raw=true)

### Step 11: Use Help and Shortcuts
- Access help menu for keyboard shortcuts (edit/command modes).

### Step 12: Explore Right Sidebar Features
- Comments, revision history, variable explorer, libraries (notebook/cluster/runtime).

### Step 13: Return to Workspace Home
- Click logo, see recent items for quick access.
